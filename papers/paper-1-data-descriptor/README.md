# Paper 1. The data descriptor

The open AI-exposure and financial-stress signal, published on its own as a
documented, reusable dataset. Descriptive. No predictive or causal claim. This
is the one we ship first, and it is the foundation Paper 2 builds on.

The steps, in order:

1. Contribution. What we release and what we claim. See `contribution.md`.
2. Build the dataset. The shared foundation: `../00-foundation/data/`.
3. Build the pipeline. The shared foundation: `../00-foundation/reproducibility/`.
4. Technical validation. Show the signal is consistent with known layoff events,
   descriptively, and that the divergence-from-baseline handles saturation.
5. Manuscript. The descriptor draft. See `manuscript.md`.
6. Submit. A data or methods venue. See `submission.md`.

The one rule: this paper makes no predictive or causal claim. It documents a
measure and releases it.
